Publicación de Alertas de tu Ciudad
Publicación y Gestión de Alertas, Avisos, Sucesos, Eventos que ocurren en tu Ciudad o cercanías.
Desarrollo de Aplicaciones Web II - Proyecto en Grupo
Requisitos Iniciales

Se desea diseñar una aplicación web para la Publicación de Alertas de tu Ciudad. La aplicación web permitirá a los usuarios visualizar y gestionar alertas, avisos, sucesos y eventos ocurridos en su ciudad o cercanías. Se podrán realizar búsquedas mediante un sistema de filtros por categorías, etiquetas, ubicación o palabras clave, mostrando los resultados en listas o mapas interactivos. Cada alerta contará con una ficha detallada que incluirá datos como título, descripción, horarios, ubicación, imágenes asociadas, estado, comentarios y valoraciones de otros usuarios.
Los usuarios registrados podrán gestionar su perfil, crear y administrar alertas, comentar, etc. Además, podrán gestionar sus publicaciones y las respuestas de moderadores o administradores.
La plataforma contará con roles diferenciados para moderadores y administradores. Los moderadores supervisarán áreas específicas, gestionando alertas y comentarios en su jurisdicción. Los administradores tendrán funciones avanzadas para gestionar categorías, etiquetas, ubicaciones y garantizar la organización y funcionalidad del sistema.

El sistema incluye una serie de funcionalidades necesarias para el manejo eficiente de alertas y usuarios:

Especificación del Diccionario de Datos

Información y Datos que debe almacenar y manejar el sistema para proporcionar las funcionalidades indicadas en los requisitos.

Elemento Descripción
Alerta Es el objeto principal de la aplicación. Contiene los datos fundamentales de un suceso, evento, aviso o alerta en la ciudad o cercanías. Cada alerta incluye:
id (Identificador único), título (Nombre o título descriptivo), descripción (Detalles sobre el evento), fecha/hora de inicio (El momento en el que se inicia la alerta), duración estimada (Duración que se espera que dure la alerta), Lugar (Ubicación geográfica, referida a la tabla Lugar), detalles (Información adicional), notas (Notas relevantes), url externa (Enlace web opcional relacionado con la alerta), Imagen principal (Imagen asociada a la alerta), [Imágenes adicionales] (Otras imágenes relacionadas), Etiquetas (Palabras clave asociadas), Clasificación (Categoría de la alerta), Estados de alerta (Estado actual de la alerta), Registro de usuario (Información del usuario que crea o modifica la alerta), Comentarios (Respuestas de usuarios).
Lugar Es la ubicación geográfica asociada a una alerta. Puede ser un punto específico en el mapa, como una calle, un barrio o una ciudad, y se relaciona con la tabla correspondiente. Este dato es crucial para filtrar alertas según la ubicación geográfica. Incluye: dirección (Dirección completa), notas del lugar (Información adicional relevante sobre el lugar), Área (Puede asociarse con áreas como País, Estado, Ciudad, etc.).
Área Define las áreas geográficas relacionadas con una alerta. Esto puede incluir categorías como país, estado, provincia, municipio, zona, etc., y se utiliza para agrupar alertas en función de la ubicación. Las áreas se organizan en jerarquías, de modo que cada área puede tener un área padre. Por ejemplo:
1=País, 2=Estado, 3=Provincia, 4=Población, 5=Zona. Además, pueden incluir códigos de área como 1=Continente, 2=País, 3=Estado, etc. Estas relaciones jerárquicas permiten realizar búsquedas filtradas por ubicación geográfica.
Imagen Elemento que representa la imagen asociada a una alerta. Se puede almacenar como un archivo externo en una carpeta específica o como un campo binario en la base de datos. La imagen puede ser una foto relacionada con el evento o una representación gráfica que ayude a ilustrar la alerta. Cada alerta puede tener una o más imágenes asociadas.
Imágenes Adicionales Enlace a las imágenes adicionales asociadas a una alerta. Este campo permite vincular múltiples imágenes a una alerta en caso de que haya más de una imagen relevante. Cada imagen adicional se relaciona a través de un identificador único y puede incluir fotos adicionales de la zona o del evento.
Etiquetas de Alerta Las etiquetas son palabras clave o términos que describen la alerta. Estas etiquetas pueden ayudar a clasificar y categorizar alertas. Por ejemplo, una alerta puede tener etiquetas como "tormenta", "accidente", "emergencia", etc.
Existen tres formas de gestionar las etiquetas:
1) Relación directa entre la Etiqueta y la Alerta.
2) Lista de identificadores de etiqueta asociada a una alerta.
3) Lista de nombres de etiquetas asociada a la alerta.
Categoría Las categorías sirven para organizar las alertas de manera jerárquica. Cada categoría tiene un nombre y una descripción, y puede tener una categoría padre. De esta forma, las alertas pueden ser clasificadas según temas específicos como "Incendios", "Inundaciones", "Accidentes", etc. Esta organización jerárquica facilita el filtrado y la gestión de las alertas.
Estados de alerta Indican el estado actual de la alerta, si está activa, visible o cerrada. Además, se registra información sobre el número de denuncias recibidas, la fecha y hora de la primera denuncia, el motivo de terminación y las razones del bloqueo de la alerta. Se pueden incluir detalles como: activa, cerrada, visible, bloqueada por denuncias o por un moderador, y el usuario bloqueador (si aplica).
Comentarios Los usuarios pueden agregar comentarios a una alerta. Cada comentario tiene un texto, un registro de usuario asociado (quién lo realizó), y un estado (abierto, cerrado). Los comentarios también pueden ser denunciados, lo que puede resultar en su bloqueo si se recibe un número determinado de denuncias. Los comentarios también pueden tener una jerarquía si son respuestas a otros comentarios.
Comentario Es una respuesta individual o reacción dentro del sistema. Cada comentario tiene un texto, un estado de cierre (abierto o cerrado), un número de denuncias, y un estado de bloqueo si ha sido reportado. Los comentarios también se asocian con el usuario que los crea y pueden tener un comentario padre si es una respuesta a otro comentario.
Registro de usuario El registro de usuario se utiliza para almacenar información sobre las acciones de los usuarios. Esto incluye datos sobre la fecha y hora de creación de un elemento (alerta, comentario), así como la fecha y hora de su modificación. Se vincula a los usuarios que crearon o modificaron una alerta o comentario, lo que permite hacer un seguimiento de las modificaciones en el sistema.
Usuario Un usuario tiene varios atributos como el email, password, nick, nombre, apellidos, dirección, fecha de registro, estado de bloqueo, etc. Los usuarios son esenciales para la creación de alertas, comentarios y la gestión de moderación. También se incluye información sobre las áreas de moderación, las denuncias y las alertas que han creado o gestionado.
Áreas de moderación Las áreas de moderación permiten a los usuarios actuar como moderadores dentro de un área geográfica específica. Un usuario puede ser asignado a una o varias áreas para gestionar alertas, comentarios y otros elementos dentro de esa zona. Las áreas pueden ser territoriales (país, estado, ciudad, etc.) y están asociadas a permisos de moderación.
Incidencia Una incidencia es un informe relacionado con una alerta o comentario. Las incidencias pueden ser de varios tipos, como avisos, notificaciones, denuncias o consultas, y se utilizan para gestionar el sistema. Las incidencias pueden tener fechas y horas de lectura, texto relacionado, y pueden estar asociadas a un Usuario (quién generó la incidencia), una Alerta o un Comentario.
Configuración La configuración es un conjunto de parámetros que definen el comportamiento y las limitaciones de la aplicación. Estos parámetros incluyen número de alertas, número de denuncias, tiempos de bloqueo, etc. Se utilizan para establecer valores predeterminados y ajustes que afectan la experiencia de usuario y las reglas de moderación.
