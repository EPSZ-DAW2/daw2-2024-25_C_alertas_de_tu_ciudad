Analisis Inicial de Funciones y Tareas

   De los requisitos iniciales planteados, se extraen una serie de funciones básicas a implementar, además de una serie de tareas adicionales a desarrollar dentro del sistema.

Roles / Perfiles:
  • Usuario Invitado (guest): Cualquier usuario que navega por las partes públicas del portal, sin necesidad de estar registrado o validado en la aplicación.
  • Usuario Registrado (normal): Pensado para los usuarios que quieren utilizar la web a la hora de crear alertas, realizar comentarrios, gestionar su perfil, gestionar sus incidencias y respuestas de los moderadores o administradores, gestionar sus propias alertas y los comentarios asociados a las alertas propias.
  • Usuario Moderador (moderador): Pensado para los usuarios que además deseen colaborar con la web a la hora de moderar las alertas y comentarios de una o varias áreas geográficas.
  • Administrador del Portal (admin): Pensado para aquellos usuarios que realizan las tareas de mantenimiento de los datos del sistema, pueden gestionar cualquier cosa de la parte privada como usuarios, alertas y datos asociados, categorías, áreas y ubicaciones, configuración, hacer copias de seguridad o restaurarlas.
  • Administrador del Sistema (sysadmin): Pensado para los programadores de la aplicación y/o administradores generales del sistema donde se instale, tendrá acceso a todas las funciones de la aplicación sin restricciones, y opcionalmente deberá poder activar el rol que desee para trabajar como un usuario de ese tipo.
